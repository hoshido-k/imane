## プロジェクト概要

### 1.1 アプリケーション名
**imane（イマネ）** - 位置情報ベース自動通知アプリ

### 1.2 コンセプト
「今ね、ここにいるよ」を自動で伝える位置情報共有アプリ。事前に目的地を設定しておくだけで、到着・滞在・退出のタイミングで大切な人に自動で通知を送信します。

### 1.3 コアバリュー
- **さりげない気配り**: 携帯を開かずに自動で位置情報を共有
- **プライバシー重視**: 必要な時だけ、選んだ相手にだけ通知
- **シンプルな体験**: チャット機能なし、通知に特化
- **自然な言葉**: 「今ね、」で始まる親しみやすいメッセージ

### 1.4 ターゲットユーザー
- パートナーや家族に自分の状況を伝えたい人
- 帰宅・到着・退出を自動で通知したい人
- 位置情報共有アプリに煩わしさを感じている人

## 機能要件

### 2.1 位置情報スケジュール管理

#### F-001: スケジュール作成
**優先度:** 高
**説明:** ユーザーが目的地と時間範囲を指定して予定を作成できる

**要件:**
- 目的地の入力方式
  - 手動入力（名前・住所）
  - 地図から選択（Google Maps API使用）
  - お気に入りから選択
- 到着判定範囲: 固定50m
- 時間指定
  - 開始時刻（必須）
  - 終了時刻（必須）
- 通知先選択
  - フレンドリストから複数選択可能
- 通知設定
  - 到着通知: ON/OFF
  - 滞在通知: ON/OFF（デフォルト60分後）
  - 退出通知: ON/OFF
- お気に入り登録: チェックボックスで設定可能

**入力バリデーション:**
- 開始時刻 < 終了時刻
- 終了時刻は現在時刻より未来
- 目的地の座標が有効範囲内
- 通知先が最低1人選択されている

---

#### F-002: スケジュール一覧表示
**優先度:** 高
**説明:** 作成したスケジュールを一覧で確認できる

**要件:**
- 表示対象: アクティブなスケジュールのみ
- 表示項目
  - 目的地名
  - 開始時刻〜終了時刻
  - 現在のステータス（active/arrived/completed/expired）
  - 通知先の人数
- ソート順: 開始時刻の昇順
- ステータス別フィルター機能

---

#### F-003: スケジュール詳細表示
**優先度:** 高
**説明:** 選択したスケジュールの詳細情報を表示

**表示項目:**
- 目的地情報（名前・住所・座標）
- 時間範囲
- 通知先一覧（ユーザー名・アイコン）
- 現在のステータス
- 到着時刻（arrived状態の場合）
- 退出時刻（departed状態の場合）
- 地図プレビュー

---

#### F-004: スケジュール更新
**優先度:** 中
**説明:** 既存のスケジュールを編集できる

**編集可能項目:**
- 時間範囲（開始・終了時刻）
- 通知先
- 通知設定（到着・滞在・退出のON/OFF）

**編集不可項目:**
- 目的地（変更する場合は新規作成）
- ステータスが completed または expired の予定

---

#### F-005: 複数スケジュール対応
**優先度:** 高
**説明:** 同時に複数の目的地を設定・トラッキング可能

**要件:**
- 時間が重複するスケジュールも作成可能
- 例: 昼は職場（12:00-18:00）、夜は飲み屋（18:00-23:00）
- すべてのアクティブスケジュールに対してジオフェンス判定を実行

### 2.3 位置情報トラッキング

#### F-201: バックグラウンド位置取得
**優先度:** 高
**説明:** アプリを閉じていても定期的に位置情報を取得・送信

**要件:**
- 更新間隔: 10分
- 取得条件: アクティブなスケジュールが存在する場合のみ
- iOS Background Location Mode使用
- 位置情報権限: "Always Allow" 必須
- バッテリー消費の最適化

**実装:**
- iOS: background_location パッケージ使用
- 位置情報取得後、即座にバックエンドへ送信

---

#### F-202: ジオフェンス判定
**優先度:** 高
**説明:** 目的地50m圏内の到着・退出を自動検知

**要件:**
- 判定ロジック
  - Haversine formula で2点間距離を計算
  - GPS誤差（±20m）を考慮した判定
  - 到着: 50m圏内に入る
  - 退出: 50m圏外に出る
- 誤検知防止
  - 連続2回の位置情報で判定（20分間）
  - 到着判定後、5分以内の退出は無視

---

#### F-203: 位置情報履歴保存
**優先度:** 高
**説明:** 位置情報を24時間TTLで保存

**要件:**
- 保存項目
  - ユーザーID
  - スケジュールID
  - 座標（緯度・経度）
  - 記録日時
  - 自動削除日時（記録日時+24時間）
- Firestore TTL機能で自動削除

---

#### F-204: トラッキングステータス表示
**優先度:** 中
**説明:** 現在のトラッキング状態を確認できる

**表示項目:**
- アクティブなスケジュール数
- 各スケジュールの現在のステータス
- 最後に位置情報を送信した時刻
- 次回位置情報取得予定時刻

### 2.4 自動通知システム

#### F-301: 到着通知
**優先度:** 高
**説明:** 目的地50m圏内到着時に自動通知を送信

**通知内容:**
```
今ね、{ユーザー名}さんが{目的地名}へ到着したよ
到着時刻: {HH:MM}
ここにいるよ → [地図リンク]
```

**トリガー条件:**
- スケジュールのステータスが active → arrived に変更
- 通知設定で「到着通知」がON
- 時間範囲内

**送信先:**
- スケジュールで指定された通知先全員

---

#### F-302: 滞在通知
**優先度:** 高
**説明:** 1時間滞在後に自動通知を送信

**通知内容:**
```
今ね、{ユーザー名}さんは{目的地名}に{X時間Y分}滞在しているよ
ここにいるよ → [地図リンク]
```

**トリガー条件:**
- 到着から60分経過
- 通知設定で「滞在通知」がON
- まだ50m圏内にいる
- 一度のみ送信（2時間後の再通知はなし）

**実装方法:**
- バックエンドのスケジューラー（Cloud Functions）で定期実行
- または、位置情報更新時に経過時間をチェック

---

#### F-303: 退出通知
**優先度:** 高
**説明:** 目的地50m圏外退出時に自動通知を送信

**通知内容:**
```
今ね、{ユーザー名}さんが{目的地名}から出発したよ
出発時刻: {HH:MM}
```

**トリガー条件:**
- スケジュールのステータスが arrived → departed に変更
- 通知設定で「退出通知」がON

---

#### F-304: 通知履歴表示
**優先度:** 中
**説明:** 過去24時間の通知履歴を確認できる

**表示項目:**
- 通知の種類（到着・滞在・退出）
- 送信元ユーザー名
- 目的地名
- 送信日時
- 地図リンク（到着・滞在通知の場合）

**フィルター:**
- 受信通知のみ
- 送信通知のみ
- すべて

---

#### F-305: 地図リンク付き通知
**優先度:** 中
**説明:** 通知に Google Maps リンクを含める

**リンクフォーマット:**
```
https://maps.google.com/?q={緯度},{経度}
```

**動作:**
- タップでGoogle Mapsアプリを起動
- iOSの場合、Apple Mapsとの選択肢を表示

## データモデル

### 4.1 LocationSchedule（位置情報スケジュール）

**コレクション名:** `location_schedules`

| フィールド名 | 型 | 必須 | 説明 |
|-------------|---|------|------|
| id | string | ✓ | スケジュールID（自動生成） |
| user_id | string | ✓ | 作成者のユーザーID |
| destination_name | string | ✓ | 目的地名 |
| destination_address | string | ✓ | 目的地住所 |
| destination_coords | map | ✓ | {lat: float, lng: float} |
| geofence_radius | int | ✓ | 到着判定半径（固定50m） |
| notify_to_user_ids | array | ✓ | 通知先のユーザーID配列 |
| start_time | timestamp | ✓ | 開始時刻 |
| end_time | timestamp | ✓ | 終了時刻 |
| recurrence | string | - | 繰り返し設定（Phase 2） |
| notify_on_arrival | bool | ✓ | 到着通知ON/OFF |
| notify_after_minutes | int | ✓ | 滞在通知タイミング（デフォルト60） |
| notify_on_departure | bool | ✓ | 退出通知ON/OFF |
| status | string | ✓ | active/arrived/completed/expired |
| arrived_at | timestamp | - | 到着時刻 |
| departed_at | timestamp | - | 退出時刻 |
| stay_notified | bool | ✓ | 滞在通知送信済みフラグ |
| favorite | bool | ✓ | お気に入り登録フラグ |
| created_at | timestamp | ✓ | 作成日時 |
| updated_at | timestamp | ✓ | 更新日時 |

**インデックス:**
- `user_id` + `status` + `start_time`（スケジュール一覧取得用）
- `status` + `end_time`（期限切れスケジュール検索用）

### 5.5 位置情報トラッキング（新規）

#### POST /api/v1/location/update
**説明:** 位置情報を送信（バックグラウンドから10分間隔で呼ばれる）

**リクエストボディ:**
```json
{
  "coords": {
    "lat": 35.689487,
    "lng": 139.691706
  },
  "accuracy": 15.5,
  "timestamp": "2025-11-04T18:35:00Z"
}
```

**処理内容:**
1. アクティブなスケジュールを取得
2. 各スケジュールに対してジオフェンス判定
3. ステータス更新（active → arrived, arrived → departed）
4. 必要に応じて通知送信
5. 位置情報履歴を保存

**レスポンス:** `200 OK`
```json
{
  "status": "ok",
  "triggered_notifications": [
    {
      "schedule_id": "schedule_123",
      "type": "arrival",
      "sent_to": ["user_xyz"]
    }
  ]
}
```

---

#### GET /api/v1/location/status
**説明:** 現在のトラッキングステータスを取得

**レスポンス:** `200 OK`
```json
{
  "active_schedules": [
    {
      "id": "schedule_123",
      "destination_name": "新宿の居酒屋",
      "status": "arrived",
      "arrived_at": "2025-11-04T18:30:00Z"
    }
  ],
  "last_location_update": "2025-11-04T18:40:00Z"
}
```

## 制約事項

### 7.1 Phase 1 での制約

| 項目 | 制約内容 |
|------|----------|
| プラットフォーム | iOS のみ（Android は Phase 2） |
| 通知チャネル | FCM のみ（LINE は Phase 2） |
| スケジュール削除 | Phase 2 で実装（Phase 1 は期限切れで自動削除のみ） |
| 繰り返しスケジュール | Phase 2 で実装 |
| 通知グループ | Phase 2 で実装（Phase 1 は個別選択のみ） |
| チャット機能 | 実装しない（通知のみに特化） |

---

### 7.2 技術的制約

| 項目 | 制約内容 |
|------|----------|
| ジオフェンス半径 | 固定50m（変更不可） |
| 位置情報更新間隔 | 固定10分（バッテリー消費考慮） |
| GPS精度 | ±20m の誤差を許容 |
| データ保持期間 | 位置情報・通知履歴は24時間のみ |
| 同時アクティブスケジュール | 制限なし（ただし、パフォーマンス考慮で推奨5個まで） |

---

### 7.3 利用制限

| 項目 | 制約内容 |
|------|----------|
| 位置情報権限 | "Always Allow" 必須（許可しない場合は機能しない） |
| ネットワーク接続 | オフライン時は位置情報をキャッシュし、接続回復時送信 |
| Firebase無料枠 | Firestoreクエリ 50,000回/日、FCM送信 無制限 |
